### 미리 준비할 프로젝트

| # | 용도 | 테스트케이스 | 피드백 | 비고 |
|---|------|-------------|--------|------|
| 1 | 피드백 많은 프로젝트 (로그인 계정 소유, 조회/수정 확인용) | 8개 (비활성화 포함) | 20개 (실패 비율 다양하게) | 최신순 정렬 시 앞에 노출 |
| 2 | 피드백 입력 시연용 프로젝트 (다른 사용자 소유) | 1개 | 없음 | 최신순 정렬 시 앞에 노출 |
| 3 | 목록 조회 + 페이지네이션용 | 자유 | 자유 | 20개 프로젝트, 만료기한 다양 (만료 8개 / 미만료 12개) |

> - 1, 2번이 최신순(기본) 정렬에서 상위 노출되도록 생성 시점 조정
> - DB 초기화 시 복구 가능하도록 MongoDB 스크립트 저장 및 공유

### 미리 준비할 환경

- 로그인 완료 상태
- **탭1**: 프로젝트 관리용 (메인 페이지)
- **탭2**: 프로젝트 생성 폼 미리 채워둔 상태 (`/projects/new`)
- **탭3**: 피드백 생성 폼 미리 채워둔 상태 (`/projects/<2번 id>/feedbacks`)
